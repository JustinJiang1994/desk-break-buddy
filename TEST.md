# 测试指南

## 快速测试步骤

### 1. 加载插件到Chrome

1. 打开Chrome浏览器
2. 访问 `chrome://extensions/`
3. 开启右上角的"开发者模式"
4. 点击"加载已解压的扩展程序"
5. 选择 `desk-break-buddy` 项目文件夹

**注意**：如果看到图标文件缺失的错误，这是正常的，功能仍然可以测试。

### 2. 测试基础功能

#### 测试快速操作面板
1. 点击浏览器工具栏的插件图标（可能显示为默认图标或错误图标）
2. 应该弹出快速操作面板
3. 检查：
   - 是否显示"起身提醒"和"喝水提醒"的状态
   - 开关是否可以切换
   - "暂停提醒"按钮是否工作
   - "进入设置"按钮是否打开设置页面

#### 测试设置页面
1. 点击"进入设置"或访问 `chrome://extensions/?options=YOUR_EXTENSION_ID`
2. 检查：
   - 提醒时间设置是否可以修改
   - 声音设置是否正常
   - 自定义计时器区域是否显示

#### 测试自定义计时器
1. 在设置页面，点击"+ 添加计时器"
2. 输入名称（如"测试提醒"）
3. 设置间隔为1分钟（用于快速测试）
4. 点击"保存"
5. 检查：
   - 计时器是否出现在列表中
   - 开关是否可以切换
   - 编辑和删除功能是否正常

#### 测试提醒功能
1. 将起身提醒间隔设置为1分钟
2. 将喝水提醒间隔设置为1分钟
3. 创建一个自定义计时器，间隔1分钟
4. 等待1分钟，检查：
   - 是否弹出提醒窗口
   - 提醒内容是否正确
   - 跳过按钮是否工作
   - 起身提醒是否30秒后自动关闭

### 3. 测试暂停功能
1. 在快速操作面板点击"暂停提醒"
2. 选择暂停15分钟
3. 检查状态是否显示"暂停中"
4. 等待暂停时间结束，检查提醒是否恢复

### 4. 测试提醒记录
1. 触发几次提醒
2. 在设置页面查看"提醒记录"区域
3. 检查是否记录了提醒历史

## 预期行为

### 正常情况
- 所有按钮和开关应该响应点击
- 设置应该能够保存
- 提醒应该按时触发
- 弹窗应该正常显示和关闭

### 已知问题（需要图标文件）
- 插件图标可能显示为默认图标或错误图标
- 这不会影响功能测试

## 调试技巧

### 查看后台脚本日志
1. 在 `chrome://extensions/` 页面
2. 找到插件，点击"service worker"链接
3. 查看控制台日志

### 查看弹窗页面日志
1. 右键点击插件图标
2. 选择"检查弹出内容"
3. 查看控制台

### 查看设置页面日志
1. 在设置页面右键
2. 选择"检查"
3. 查看控制台

## 常见问题

**Q: 插件加载失败？**
A: 检查 manifest.json 是否有语法错误，查看扩展程序页面的错误信息

**Q: 提醒不触发？**
A: 
- 检查是否开启了提醒功能
- 检查是否在暂停期间
- 查看后台脚本控制台是否有错误

**Q: 自定义计时器不显示？**
A: 
- 检查是否保存了设置
- 查看浏览器控制台是否有JavaScript错误
